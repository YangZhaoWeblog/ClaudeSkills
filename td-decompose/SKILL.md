# 学习粒度诊断器

learn-deep 的上游诊断器。**learn-deep 粒度 = 一个能在一次对话中走完"缺口→方案→新缺口"发明之路的知识概念。**

- **一个发明故事** → 不拆，建议直接 learn-deep（路径 A）
- **多个并行故事** → 拆，输出 DAG（路径 B）

DAG 是诊断结果之一，不是默认输出。**能不拆就不拆。**

## 标定案例

| 输入 | 判定 | 理由 |
|------|------|------|
| 数字证书 | **不拆** | 一条连贯缺口链：身份认证→签名→公钥分发→CA→信任锚→预装根证书→… |
| Paxos | **不拆** | 单一发明路：一致性缺口→多数派→提案冲突→两阶段→活锁→… |
| 快速排序 | **不拆** | 单一发明路：排序→分治→pivot选择→partition→复杂度→… |
| 分布式系统 | **拆** | 并行多线：一致性、容错、分区、复制、共识各自独立 |
| Web安全 | **拆** | 并行多域：XSS、CSRF、注入、认证各自独立 |
| 操作系统 | **拆** | 并行多线：进程、内存、文件系统、IO 各自独立 |

用这些案例做类比锚点，不要从零推理。

## 执行流程

### 第 1 步：快速摸底

> "你对{主题}的了解程度？（零基础 / 知道一些 / 有基础想补漏）"

用户不回答按"知道一些"处理。同时用 obsidian CLI 扫描 vault 已有知识：
- `obsidian search query="{关键词}" path="1.原子笔记"`
- `obsidian backlinks file="{概念名}"`

### 第 2 步：粒度诊断（必须显式输出，不可跳过）

**写一条缺口链**：`{缺口} → {方案} → {新缺口} → ...`，单行箭头，不换行不解释。

- **一条不分叉的链**（1-2 个可剥离支线不算分叉）→ 单故事，走路径 A
- **3+ 条无法合并的并行线索** → 多故事，走路径 B

必须把缺口链和判定结果显式输出。这是诊断证据，不是内心独白。

### 路径 A：单故事 → 不拆（~10 行）

```
{主题} · 粒度诊断
判定：单故事 ✅ — 建议直接 learn-deep

缺口链：{缺口} → {方案} → {新缺口} → ...
已有知识°：{vault 中已有的相关概念}
路径上下文：
- 已有基础：{vault 中已有原子笔记的概念}
- 重点关注：{一句话——建议从什么角度切入}

下一步：说"学透 {主题}"开始深学
```

### 路径 B：多故事 → 拆 → 输出 DAG（~25 行）

```
{主题} · 知识地图
> {一句话学习目标}

缺口链测试：检测到 N 条并行线索 → 需要拆分

{ASCII DAG 图，每个节点名后紧跟标色。示例：}
{  概念A°──→ 概念B🔴 ──→ 概念C🟡  }
{       └──→ 概念D🔴⚠ ──→ 概念E🟢  }

已有知识°：{vault 中已有的相关概念，标°}
🔴预检：{节点A→✅单线} | {节点B→⚠3线(子题1/子题2/子题3)} | ...
🔴 = deep-learn  🔴⚠ = deep-learn（需先拆分）  🟡 = 快速阅读  🟢 = 动手
★ 枢纽：{节点(N↓)}  入口：{节点列表}

路线：① {节点}🟡 → ② {节点}🔴 → ③ {节点}🔴⚠（先拆分） → ...

🔴 为什么深学：
- {节点A} — {一句话理由}

下一步：说"学透 {第一个🔴节点}"开始深学 | 追问任意节点 | 调整结构 | 说"确认"保存
```

**节点定义**：一个 learn-deep 能一次走完发明之路的知识概念。不是学习动作，不是技能，不是章节标题。

**标色逻辑**：高杠杆+高可建构→🔴候选 → 速读后学下游会卡住吗？卡→🔴，不卡→🟡。实践性强→🟢。

### 🔴 节点粒度预检（路径 B 必须执行）

对每个🔴节点跑一次单行缺口链测试。**对抗性思考：刻意尝试找分叉——这个节点是否包含 3+ 个可以独立讲完的子发明？** 不分叉→确认 `🔴`。分叉 3+→标记 `🔴⚠`，旁注"含 N 条并行线索，到站后拆分"。

`🔴⚠` 节点保留在 DAG 中不展开。用户说"学透 X"时自动触发局部 decompose 输出子 DAG。

## 路径上下文（decompose → learn-deep 接口）

用户说"学透 {节点}"时，自动生成：

```
学透 {节点名}
路径上下文（来自 decompose）：
- 学习路径："{大主题}" | 已深学：{已完成🔴} | 已速读：{已完成🟡}
- 已有基础：{vault 已有原子笔记} | 下游：{DAG 下游节点}
- 重点关注：{一句话——为什么在这条路上学这个，从什么角度切入}
```

## 迭代（仅路径 B）

- 追问节点 → 展开节点卡片（定义+评估+理由）
- "X 太粗了" / "X 和 Y 合并" / "X 应该标红" → 调整后重新输出**完整 DAG**
- 每次调整后重新输出完整 DAG（不只差异）

{单行缺口链}

- 重点关注：{切入角度}

## 质量护栏

1. **粒度诊断不可跳过**——必须先输出缺口链和判定，再决定路径
2. **两条路径互斥**——不拆时不输出 DAG，拆时不输出诊断报告
3. **节点数 ≤ 15**，无孤儿节点（每节点至少一条边）
4. **🔴 节点必须有一句话理由**
5. **用户点名的概念必须是独立节点**——不默默吞掉
6. **终端输出 ≤ 25 行**（不含已有知识行）
7. **不在草案阶段写文件**——用户说"确认"后才写
