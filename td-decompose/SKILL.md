# 学习粒度诊断器

learn-deep 的上游诊断器。**learn-deep 粒度 = 一个能在一次对话中走完"缺口→方案→新缺口"发明之路的知识概念。**

- **一个发明故事** → 不拆，建议直接 learn-deep（路径 A）
- **多个并行故事** → 拆，输出 DAG（路径 B）

DAG 是诊断结果之一，不是默认输出。**能不拆就不拆。**

## 标定案例

| 输入 | 判定 | 理由 |
|------|------|------|
| 数字证书 | **不拆** | 一条连贯缺口链：身份认证→签名→公钥分发→CA→信任锚→预装根证书→… |
| Paxos | **不拆** | 单一发明路：一致性缺口→多数派→提案冲突→两阶段→活锁→… |
| 快速排序 | **不拆** | 单一发明路：排序→分治→pivot选择→partition→复杂度→… |
| 分布式系统 | **标🔴⚠** | 并行多线：一致性、容错、分区、复制、共识各自独立 |
| Web安全 | **标🔴⚠** | 并行多域：XSS、CSRF、注入、认证各自独立 |
| 操作系统 | **标🔴⚠** | 并行多线：进程、内存、文件系统、IO 各自独立 |
| 共识机制 | **标🔴⚠** | 并行多线：PoW、PoS、BFT 各自独立发明故事 |
| 密码学 | **标🔴⚠** | 并行多线：对称加密、公钥密码、哈希函数各自独立 |

用这些案例做类比锚点，不要从零推理。**"标🔴⚠"意味着该概念必须作为一个🔴⚠节点保留在 DAG 中，不要帮用户拆分——由用户说"拆分 X"时再展开。**

## 执行流程

### 第 1 步：快速摸底

> "你对{主题}的了解程度？（零基础 / 知道一些 / 有基础想补漏）"

用户不回答按"知道一些"处理。同时用 obsidian CLI 扫描 vault 已有知识：
- `obsidian search query="{关键词}" path="1.原子笔记"`
- `obsidian backlinks file="{概念名}"`

### 第 2 步：粒度诊断（内部执行，不输出给用户）

写一条缺口链，判断单故事/多故事。**诊断过程是内部证据，不展示给用户。** 用户只看到最终的路径 A 或路径 B 输出。

### 路径 A：单故事 → 不拆（~10 行）

```
{主题} · 粒度诊断
判定：单故事 ✅ — 建议直接 learn-deep

缺口链：{缺口} → {方案} → {新缺口} → ...
已有知识°：{vault 中已有的相关概念}
路径上下文：
- 已有基础：{vault 中已有原子笔记的概念}
- 重点关注：{一句话——建议从什么角度切入}

下一步：说"学透 {主题}"开始深学
```

### 路径 B：多故事 → 拆 → 输出 DAG（~10 行）

```
{主题} · 知识地图（已有：{已有概念°列表}）

{ASCII DAG 图，每个节点名后紧跟标色。示例：}
{  概念A°──→ 概念B🔴 ──→ 概念C🟡  }
{       └──→ 概念D🔴⚠ ──→ 概念E🟢  }

路线：① {节点}🔴（一句话定义） → ② {节点}🔴⚠（一句话定义） → ③ {节点}🟡 → ...

下一步：说"学透 {第一个🔴节点}"开始 | 说"拆分 {⚠节点}"展开子节点 | 追问任意节点 | 说"确认"保存
```

首屏只输出 DAG 图 + 路线 + 下一步。**🔴理由块追问时再展开，不在首屏堆砌。** 节点一句话定义可内嵌路线或用简表呈现。

🔴 = deep-learn  🔴⚠ = deep-learn（需先拆分）  🟡 = 快速阅读  🟢 = 动手

**节点定义**：一个 learn-deep 能一次走完发明之路的知识概念。不是学习动作，不是技能，不是章节标题。

**标色逻辑**：高杠杆+高可建构→🔴候选 → 速读后学下游会卡住吗？卡→🔴，不卡→🟡。实践性强→🟢。

**渐进披露**：用户追问节点时展开节点卡片（定义+评估+🔴理由）。不在首屏堆砌。

## 路径上下文（decompose → learn-deep 接口）

用户说"学透 {节点}"时，自动生成：

```
学透 {节点名}
路径上下文（来自 decompose）：
- 学习路径："{大主题}" | 已深学：{已完成🔴} | 已速读：{已完成🟡}
- 已有基础：{vault 已有原子笔记} | 下游：{DAG 下游节点}
- 重点关注：{一句话——为什么在这条路上学这个，从什么角度切入}
```

## 迭代（仅路径 B）

- 追问节点 → 展开节点卡片（定义+评估+理由）
- "X 太粗了" / "X 和 Y 合并" / "X 应该标红" → 调整后重新输出**完整 DAG**
- 每次调整后重新输出完整 DAG（不只差异）

## 完成后：粒度审查

DAG 输出后，**必须**启动独立审查 Agent：

```
你是 decompose 的粒度审查员。

输入：一张 DAG，其中标🔴的节点声称"learn-deep 可以一次走完"。

任务：两项检查。

检查一·粒度：对每个🔴节点，按以下三步判断：
1. 穷举：尽可能多地列出该节点的子主题（不怕列多）
2. 合并：哪些子主题是同一条"问题→方案"链上的连续步骤？合并它们
3. 判定：合并后剩余独立链 ≤ 2 → ✅ 确认🔴；≥ 3 → ⚠ 建议改🔴⚠

检查二·完整性：DAG 中是否存在已知多故事概念被拆散（只保留了部分子节点，遗漏了主流替代方案）？
如果某个概念的多个子发明中只有一个出现在 DAG 里，其余消失了，报告遗漏。

输出 JSON：
{
  "granularity": [{"node": "节点名", "subs": ["子发明1", "子发明2", "子发明3"], "suggest": "🔴⚠"}],
  "completeness": [{"missing_parent": "概念名", "present": ["已有子节点"], "missing": ["遗漏子节点"]}]
}
无问题则对应数组为 []
```

收到审查结果后，将🔴⚠标记回写到 DAG 图和路线中，输出最终版本给用户。

## 大主题判定（仅路径 B，粒度审查后执行）

DAG 通过粒度审查后，做一次**结构性判定**，决定写入文件时用**扁平模板**还是**分层模板**。

**判据**（全部满足 = 大主题 → 分层模板）：

- ✓ 可自然划分出 **≥ 3 个领域组**
- ✓ 每个组内 **≥ 3 个节点**
- ✓ 至少 1 个组是"硬前置"或"主干入口"（区别于旁支）

任一条件不满足 → 扁平模板（v1 原行为）。

**用结构判据，不用节点数字**——LLM 数数字不稳，判断结构性质稳。

**标定案例**（类比锚点，不从零推理）：

| 主题 | 判定 | 理由 |
|------|------|------|
| 数字证书 | ❌ 扁平 | 单条连贯链，无 3 组划分 |
| 共识算法 | ❌ 扁平 | 7 节点、无硬前置组 |
| 红黑树 | ❌ 扁平 | 节点太少 |
| 分布式系统 | ✅ 分层 | 一致性/容错/共识/复制 多组独立 |
| 操作系统 | ✅ 分层 | 进程/内存/文件系统/IO，硬前置明显 |
| 价值投资 | ✅ 分层 | 财报硬前置 + 6 天然分组 |
| 密码学 | ✅ 分层 | 对称/公钥/哈希/协议 多组独立 |
| Web 安全 | ✅ 分层 | XSS/CSRF/注入/认证 多组独立 |

判定结果用于"写入文件"阶段选择模板。

## 写入文件（用户说"确认"或"保存"后）

路径：`0.Inbox/学习-{主题}/DAG-{主题}.md`（用 Bash mkdir -p 创建目录）。全文概念名用 `[[wikilink]]`。

**根据大主题判定结果选择模板**：单故事 → A 模板；多故事 + 扁平 → B 模板；多故事 + 分层 → C 模板。

### 模板 A：单故事

```markdown
---
tags: [学习DAG, {主题关键词}]
created: {YYYY-MM-DD}
---

# {主题} · 知识分解图

> 判定：单故事 ✅

## 缺口链

{单行缺口链}

## 路径上下文

- 已有基础：{已有原子笔记概念}
- 重点关注：{切入角度}
- 可选延伸：{支线话题，如有}

下一步：`学透 [[{主题}]]`
```

### 模板 B：多故事 · 扁平（小/中主题）

```markdown
---
tags: [学习DAG, {主题关键词}]
created: {YYYY-MM-DD}
---

# {主题} · 知识分解图

> 判定：多故事
> 学习目标：{一句话}

## 知识地图

{ASCII DAG 图 + 图例}

## 路线

① [[节点1]]🔴 → ② [[节点2]]🔴⚠ → ③ [[节点3]]🟡 → ...

- [ ] **[[{节点名}]]** 🔴
      {一句话定义}
      依赖：[[节点A]]、[[节点B]]

### 模板 C：多故事 · 分层（大主题，精简长期版）

```markdown
---
tags: [学习DAG, {主题关键词}]
created: {YYYY-MM-DD}
---

# {主题} · 知识分解图

> 判定：多故事 · 大主题
> 学习目标：{一句话}
> 已有°：{vault 已有原子笔记}

图例：🔴 deep-learn　🔴⚠ 需拆分　🟡 快速阅读　🟢 动手　° 已有原子笔记

---

## 一、首屏总览（{N} 个领域组）

{紧凑 ASCII 方框图。每组一个方框，含：
 - 组名 + 颜色 + 节点数 + 标签（硬前置/主干入口/主干/旁支）
 - 组的一行描述
 - 组间依赖用箭头连接，箭头旁可标注传递语义}

推荐顺序：{例：A → B → C → D → E}，{旁支组} 随时可插入。

---

## 二、完整学习路径

按组分段，每节点 checkbox + 一行极简定义。打勾追踪进度。

### A. {组名}（{标签}）

- [ ] [[节点1]]🔴 — {定义 ≤ 15 字}
- [ ] [[节点2]]🔴 — {定义 ≤ 15 字}
...

### B. {组名}（{标签}）

- [ ] [[节点3]]🔴 — {定义 ≤ 15 字}
...

（每组一小节，所有节点必须在此出现，包括旁支）

---

## 三、组内结构图

每组一张 ASCII 小图，只显示组内依赖和跨组入口/出口，无文字叙述。

### A. {组名}

```text
{组内 ASCII DAG 图}
```

### B. {组名}

```text
{组内 ASCII DAG 图}
```

...

```

**分层模板核心要点**：

1. **首屏只出组级 ASCII 方框图**，不铺具体节点
2. **完整路径包含所有节点**（主干 + 旁支），用 checkbox 追踪长期进度
3. **每节点一行定义**（见"定义压缩规则"），不再有单独的"🔴 理由"字段
4. **组内结构图只留 ASCII**，删去"出口"等文字叙述
5. **无"📖 如何使用"、"完整节点清单"、"审视记录"**——DAG 是图不是手册

---

### 定义压缩规则（模板 C 使用）

节点定义必须压到**一行 ≤ 15 字**，服务长期扫视识别。

**格式**：`[关键概念] + [属性/动作] + [形容词]`，用 `—` 破折号连接节点名。

**写法**：

- 无需完整句子
- 可用 `/` `-` `+` 等符号连接
- 15 字是上限，10 字更佳
- 如原定义已 ≤ 15 字，直接用

**示例**：

| 节点 | ✅ 压缩版（≤ 15 字） | ❌ 过长版 |
|------|---------------------|----------|
| 资产负债表 | 某时点的资产/负债/权益快照 | 公司在某个时点拥有什么、欠什么、净剩什么的快照 |
| 利润表 | 某时段的收入-成本-利润流 | 公司在一段时间内怎么赚钱的收入、成本、利润流 |
| 市场先生 | 情绪化报价员，价格常偏离 | 市场是一个情绪化的报价员，价格会持续偏离价值 |
| 护城河五力与定价权 | 5 种壁垒 + 定价权外显 | 品牌/转换成本/网络效应/成本优势/特许经营五种机制... |


## 质量护栏

1. **粒度诊断不可跳过**——必须先输出缺口链和判定，再决定路径
2. **两条路径互斥**——不拆时不输出 DAG，拆时不输出诊断报告
3. **无孤儿节点**——每节点至少一条边（依赖或被依赖）
4. **🔴 节点必须有一句话理由**（追问时展开）
5. **用户点名的概念必须是独立节点**——不默默吞掉
6. **不在草案阶段写文件**——用户说"确认"后才写
7. **粒度审查 Agent 不可跳过**——DAG 输出后必须跑审查再展示最终版
8. **大主题判定不可跳过**——写文件前必须用结构判据选模板（扁平/分层），不用节点数字
