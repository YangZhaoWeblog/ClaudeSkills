# 彻底搞懂 RSA 算法

## 一、RSA 要解决什么问题？

想象一个场景：你想在网上给朋友发一条秘密消息，但你们之间的通信线路可能被人窃听。传统的加密方法（对称加密）要求你们事先共享一把"密钥"——但问题是，这把密钥本身怎么安全地传过去？

RSA 的天才之处在于：**它让你可以公开告诉全世界你的"加密钥匙"（公钥），但只有你自己才能解密（用私钥）。** 这就像一个任何人都能锁上的信箱，但只有你有钥匙能打开它。

这就是"非对称加密"的核心思想，而 RSA 是第一个被广泛使用的非对称加密算法，由 Rivest、Shamir 和 Adleman 三人在 1977 年提出。

## 二、RSA 的数学基础

### 2.1 质数（素数）

质数是只能被 1 和自身整除的大于 1 的自然数。比如 2, 3, 5, 7, 11, 13...

质数有一个关键性质：**两个大质数相乘很容易，但把一个大数分解回两个质数极其困难。**

比如：
- 计算 61 x 53 = 3233 —— 一秒钟的事
- 反过来，给你 3233，让你找出它是哪两个质数的乘积 —— 对小数字还行，但如果数字有几百位，现有计算机几乎无法完成

这就是 RSA 安全性的根基：**大整数分解的计算困难性。**

### 2.2 模运算（取余）

模运算就是除法取余数：
- 17 mod 5 = 2（因为 17 = 3 x 5 + 2）
- 23 mod 7 = 2（因为 23 = 3 x 7 + 2）

模运算有一个重要特性：它是一种"单向"运算。知道 `x mod n = r`，你很难反推出 x 是多少（因为有无穷多个 x 满足条件）。

### 2.3 欧拉函数 φ(n)

欧拉函数 φ(n) 表示从 1 到 n 中，与 n 互质（最大公约数为 1）的数的个数。

关键性质：
- 如果 p 是质数，φ(p) = p - 1（因为质数与比它小的所有正整数都互质）
- 如果 n = p x q（p、q 都是质数），那么 **φ(n) = (p-1)(q-1)**

这个公式非常关键！它意味着：**如果你知道 n 的质因数分解，计算 φ(n) 很简单；但如果你不知道分解，计算 φ(n) 就和分解 n 一样难。**

### 2.4 欧拉定理

如果 a 和 n 互质，那么：

```
a^φ(n) ≡ 1 (mod n)
```

也就是说，a 的 φ(n) 次方除以 n，余数一定是 1。

由此推导：

```
a^(k·φ(n) + 1) ≡ a (mod n)
```

这意味着：**如果你对 a 做一个特殊幂运算，再取模，可以把 a "还原"回来。** 这就是 RSA 能"加密再解密"的数学原因。

## 三、RSA 算法完整流程

### 第一步：生成密钥

1. **选两个大质数** p 和 q（实际应用中每个都有 1024 位以上）
2. **计算 n = p x q**（这个 n 就是公开的模数）
3. **计算 φ(n) = (p-1)(q-1)**
4. **选一个公钥指数 e**，要求：
   - 1 < e < φ(n)
   - e 与 φ(n) 互质
   - 通常选 e = 65537（因为它是质数，且二进制表示中 1 很少，计算快）
5. **计算私钥指数 d**，使得：
   - e x d ≡ 1 (mod φ(n))
   - 也就是说 d 是 e 在模 φ(n) 下的"乘法逆元"
   - 可以用扩展欧几里得算法高效计算

最终：
- **公钥 = (n, e)** —— 公开给所有人
- **私钥 = (n, d)** —— 自己秘密保存
- p, q, φ(n) 全部销毁，不能泄露

### 第二步：加密

假设要加密的消息是一个数字 m（0 <= m < n），加密计算：

```
c = m^e mod n
```

其中 c 是密文。任何人都可以用公钥 (n, e) 来加密。

### 第三步：解密

收到密文 c 后，用私钥解密：

```
m = c^d mod n
```

这样就恢复了原始消息 m。

### 为什么这样能还原？

```
c^d mod n
= (m^e)^d mod n
= m^(e·d) mod n
```

因为 e·d ≡ 1 (mod φ(n))，也就是说 e·d = k·φ(n) + 1，所以：

```
m^(e·d) mod n
= m^(k·φ(n) + 1) mod n
= m  （根据欧拉定理的推论）
```

完美还原！

## 四、一个完整的数字例子

为了直观理解，我们用很小的数字走一遍：

### 生成密钥

1. 选 p = 61, q = 53
2. n = 61 x 53 = **3233**
3. φ(n) = 60 x 52 = **3120**
4. 选 e = 17（17 与 3120 互质 ）
5. 求 d，使 17 x d ≡ 1 (mod 3120)
   - d = **2753**（因为 17 x 2753 = 46801 = 15 x 3120 + 1）

- 公钥：(3233, 17)
- 私钥：(3233, 2753)

### 加密

要加密消息 m = 65（比如字母 'A' 的 ASCII 码）：

```
c = 65^17 mod 3233 = 2790
```

密文是 2790。

### 解密

```
m = 2790^2753 mod 3233 = 65
```

成功还原了 65！

## 五、RSA 为什么安全？——回答你的核心疑问

你的问题是："为什么用两个大质数就能加密？"

答案的核心在于**一个不对称性**：

| 操作 | 难度 | 谁能做 |
|------|------|--------|
| p x q → n | 极易（一次乘法） | 任何人 |
| n → p, q | 极难（大整数分解） | 无人能做（数字足够大时） |
| 知道 p, q → 算 φ(n) → 算 d | 容易 | 密钥生成者 |
| 只知道 n, e → 算 d | 极难 | 无人能做 |

攻击者知道的是公钥 (n, e)。要破解 RSA，他需要求出 d。要求 d，他需要知道 φ(n)。要知道 φ(n)，他需要把 n 分解为 p x q。而**大整数分解是一个没有已知高效算法的问题**。

目前世界纪录：2020 年，研究者分解了一个 829 位（250 个十进制数字）的数，用了约 2700 个 CPU 核心年的算力。而现代 RSA 使用的是 2048 位甚至 4096 位的密钥——分解这样大的数字，以目前的技术，需要的时间远超宇宙年龄。

### 打个比方

RSA 就像一个这样的机关：
- **制造锁**：把两种特殊颜料混合（质数相乘），混合后你得到了一种新颜色（n）。你把这个新颜色公布给全世界（公钥）。
- **加密**：任何人都可以用这个颜色把消息"涂上色"（幂运算取模）。涂上色后，消息变得不可辨认。
- **解密**：只有你知道原始的两种颜料是什么（p 和 q），所以只有你能"褪色"（计算私钥 d 来解密）。
- **安全性**：从混合后的颜色反推出原始两种颜料，是极其困难的（大数分解难题）。

## 六、RSA 的实际应用

### 6.1 数字签名

RSA 不仅能加密，还能做数字签名（证明消息确实是你发的）：
- **签名**：用你的私钥对消息的哈希值加密 → 得到签名
- **验证**：任何人用你的公钥解密签名 → 得到哈希值，与消息的实际哈希值比对

### 6.2 HTTPS / TLS

你每次访问 https:// 网站时，浏览器和服务器之间会用 RSA（或类似算法）交换一个对称加密的密钥，然后用对称加密传输数据。RSA 在这里起到的是"安全地交换密钥"的作用。

### 6.3 实际使用中的注意事项

- RSA 加密比对称加密慢几百到几千倍，所以实际中通常只用 RSA 加密一个对称密钥，再用对称密钥加密大量数据
- 直接对消息做 RSA 加密需要"填充"（padding），如 OAEP，否则会有安全漏洞
- 密钥长度至少 2048 位，推荐 4096 位

## 七、RSA 的潜在威胁

### 量子计算

1994 年，数学家 Peter Shor 提出了 Shor 算法，理论上量子计算机可以在多项式时间内分解大整数。如果大规模量子计算机实现，RSA 将被破解。因此，密码学界正在研发"后量子密码"（Post-Quantum Cryptography）来替代 RSA。

但截至目前，能破解实际 RSA 密钥的量子计算机尚不存在。

## 八、总结

RSA 的精髓可以归结为一句话：

> **乘法容易，分解难。知道因子就能解密，不知道因子就无法解密。**

整个算法建立在三个支柱上：
1. **大整数分解的困难性** —— 安全性来源
2. **欧拉定理** —— 数学上保证加密后能正确解密
3. **模幂运算的高效性** —— 计算上保证加密和解密是可行的

这就是为什么"两个大质数"能实现加密：不是质数本身有什么魔法，而是**"相乘容易、分解难"这个数学事实**，构造出了一道只有持有密钥的人才能通过的单向门。
