### `[修改]` 涌现洞察 → 追加到"暗线：贯穿全程的两条线"之后

**三个 EC 方案的隐藏统一结构：同一把锤子的三种握法**

| 洞察 | 说明 |
|------|------|
| **"随机数 k"是整条 EC 线的命门** | ECDH 的安全靠私钥不泄露，ECDHE 靠临时私钥用完即毁，ECDSA 靠 $k$ 绝不重复。三者的灾难模式全部归结为"某个随机数被知道了"——随机数生成器是椭圆曲线密码学的单点故障。 |
| **ECDHE + ECDSA 是同一次握手里的"左右手"** | TLS 1.3 握手中，ECDHE 生成的临时密钥对用于协商秘密，服务端立即用 ECDSA 签名这个临时公钥证明身份。两者数学地基相同（ECDLP），但一个输出共享秘密、一个输出签名——协商与认证在同一条曲线上完成，密钥对可以复用同一套曲线参数（如 P-256），这是 ECC 相比 RSA+DH 组合的结构性优势。 |
| **ECDH → ECDHE 的演化揭示了密码学的通用升级模式** | 静态 → 临时，是密码学反复使用的"加一层随机性换前向保密"套路。DH→DHE、ECDH→ECDHE、甚至未来后量子方案（Kyber 也有临时变体），同一个模式在不同数学地基上反复重演。识别这个模式，可以预测新方案的演化方向。 |
