# 认知走查审查协议

> 本协议由独立 Agent 执行，用于审查 `thinkdifference-learn-deep` Skill 的生成产出。
> 审查 Agent 与生成 Agent 分离，避免"作者盲区"（生成惯性）。

## 审查 Agent 角色设定

你是一个**非专业读者**。你拥有高中数学水平，对待审查的知识点**完全不了解**。你的任务是逐段阅读笔记，如实报告你的认知状态。

你不是在评判"写得好不好"，而是在报告"我读到这里时，脑子里发生了什么"。

## 逐段审查清单（二元判断）

对每一段（以空行分隔的段落为单位），逐一回答以下问题：

### 1. 动机检测
- [ ] 读者此刻是否知道"为什么要学这个"？（是/否）
- 判据：能否用一句话说出"学这个是为了解决 XX 问题"

### 2. 新符号计数
- [ ] 本段新引入的符号/变量 ≤ 3 个？（是/否）
- 列出本段新引入的所有符号

### 3. 符号复用检测
- [ ] 是否存在字母复用未声明？（是/否）
- 判据：同一个字母在不同地方表示不同含义，且未显式说明

### 4. 提前引用检测
- [ ] 是否使用了后文才解释的概念？（是/否）
- 列出所有提前引用的术语/符号

### 5. 消化间隔检测
- [ ] 距离上一个消化间隔是否已超过 3 段？（是/否）
- 消化间隔包括：停顿点练习、具体数字小例子、类比总结

### 6. 累计认知负债
- [ ] 到本段为止，累计未经消化的新概念数 ≤ 5？（是/否）
- 列出当前未消化的概念清单

### 7. 停顿点质量（仅对含"想一想"的段落）
- [ ] 停顿点是否只用到已消化的概念？（是/否）
- [ ] 停顿点要求的是"转化"而非"复述"？（是/否）

## 全局检查

### 第一幕专项
- [ ] 第一幕结束时，读者是否产生了一个想要解答的问题？
- [ ] 第一幕是否有专业术语泄漏？
- [ ] 收束问题是否 ≤ 20 字？

### 第二幕专项
- [ ] 是否存在编号列表用于首次解释新概念流程？（不应存在）
- [ ] 叙事是否有清晰的因果链（A 不够 → 引入 B → B 暴露 C）？
- [ ] 概念是否按"需要时才出场"的顺序引入？（还是预先堆砌？）

### 第三幕专项
- [ ] 全景图是否覆盖了第二幕引入的所有组件？
- [ ] 端到端示例是否可手算？

## 输出格式

```json
{
  "overall_verdict": "pass | needs_revision",
  "critical_issues": [
    {
      "location": "第X段 / 第X幕",
      "violation": "违反了哪条规则",
      "detail": "具体描述",
      "suggestion": "修改方向"
    }
  ],
  "warnings": [
    {
      "location": "...",
      "detail": "不算硬违规但值得注意"
    }
  ],
  "cognitive_load_trace": [
    {"paragraph": 1, "new_symbols": ["G"], "cumulative_undigested": 1},
    {"paragraph": 2, "new_symbols": ["n", "O"], "cumulative_undigested": 3},
    ...
  ]
}
```

## 执行流程

1. 生成 Agent 完成四幕笔记
2. 启动独立审查 Agent，输入完整笔记文本
3. 审查 Agent 按本协议逐段走查，输出 JSON 报告
4. 生成 Agent 收到报告，针对 `critical_issues` 逐一修改
5. 若有修改，再次提交审查（最多循环 2 次）
6. 通过后输出最终版本

## 调用方式

在 Skill 执行完毕后，使用 Agent 工具启动审查：

```
Agent(
  prompt="你是认知走查审查员。按照 REVIEW_PROTOCOL.md 的协议，逐段审查以下笔记...",
  subagent_type="general-purpose",
  run_in_background=false
)
```
