# 认知走查审查协议

> 本协议由独立 Agent 执行，用于审查 `thinkdifference-learn-deep` Skill 的生成产出。
> 审查 Agent 与生成 Agent 分离，避免"作者盲区"（生成惯性）。

## 审查 Agent 角色设定

你是一个**文本审查员**，不是读者模拟器。你的任务是对笔记执行两类检查：

- **第一轮：机械检查**——数数、找字、匹配模式。这类判断不需要理解内容，可靠度高
- **第二轮：结构判断**——评估叙事逻辑。这类判断需要主观评估，结果标记为"建议"而非"违规"

## 第一轮：机械检查（硬违规）

对全文逐段执行，每项只需 是/否：

### M1. 符号密度
每段（空行分隔）新引入的数学符号/变量名 ≤ 3 个？
- 计数方法：只算**首次出现**的符号，已出现过的不算
- 违规示例：一段内同时出现 $p$, $q$, $n$, $\varphi$, $e$ → 5 个，违规

### M2. 符号复用
同一字母在不同位置表示不同含义时，是否有显式声明？
- 查找方法：扫描全文，找到同一字母被重复定义的位置
- 违规示例：$k$ 先表示"走几步"，后表示"对称密钥"，无声明 → 违规

### M3. 连续灌输
是否存在连续 3 段以上都没有消化间隔？
- 消化间隔的标志：`> 想一想`、具体数字小例子、类比一句话总结
- 计数方法：逐段扫描，记录"距上一个消化间隔的段落数"

### M4. 列表误用
第二幕中是否出现编号列表（`1. 2. 3.`）用于首次解释一个流程型/因果型新概念？
- 第三幕全景走查中的编号列表允许存在
- 若新概念本质上是在回答"它由什么组成"，允许使用 2-4 条短列表辅助澄清，但不得替代前后缺口驱动的叙事
- 违规示例："Alice 做三件事：1. 生成 r 2. 计算 S 3. 加密" → 违规

### M5. 第一幕术语泄漏
第一幕中是否出现专业术语（未经解释直接使用）？
- 违规示例：第一幕开篇出现"单向函数"、"模运算"、"公钥" → 违规

### M5b. 第一幕假因果接入
第一幕是否从前一篇 deeplearn 的开放缺口硬接入，而该缺口与本知识点之间不存在真实因果关系？
- 检验方法：把前一篇删掉，本篇第一幕的问题还成不成立？成立 = 假因果，违规
- 违规示例：前一篇区块链末尾留下"重做区块为何够贵→共识机制"，下一篇系统模型开头写"上一站你搭完了区块链...但重做太便宜...所以需要定义系统模型" → 系统模型的问题不依赖区块链，删掉区块链照样成立，属于假因果接入
- 正确做法：第一幕从知识点自身的原生问题独立构建。DAG 上下文仅用于：(1) 判断读者知识现状（已学概念可直接引用）(2) 第四幕开放缺口指向

### M6. 等式链过长（数学层·含推导链时检查）
单行等式中是否出现超过 2 个等号？
- 违规示例：`$a = b = c = d$` → 3 次变换，违规
- 正确做法：拆成多行，每行只做一次变换

### M7. 等式变换缺少操作说明（数学层·含推导链时检查）
等式变换是否缺少"这步在做什么"的说明？
- 正确格式：操作说明在前，等式在后
- 违规示例：连续两个等式，中间无任何文字说明变换依据

### M8. 结论缺少机制
结论性陈述是否只给了验证例子而没有给出机制（为什么成立）？
- 判据：把例子删掉，剩下的文字能否独立解释结论？能 → 合格；不能 → 违规
- 若机制不存在（设计决策、经验参数），必须显式声明"这是设计选择，不是数学必然"

### M9. 类比缺少脱壳
使用了类比/比喻后，是否缺少脱壳动作？
- 脱壳 = 说明（a）类比对应真实机制中的什么，（b）类比从哪里开始不成立
- 查找方法：扫描全文中的类比/比喻表述，检查其后是否有对应说明和失效边界
- 第一幕场景类比：在本幕结尾或第二幕开头须有过渡回真实机制的动作
- 第二幕消化间隔中的一句话类比：紧跟其后须有简短脱壳

### M10. 端到端示例心算律
第三幕端到端示例中的数值计算是否可心算？
- 违规条件：单步乘法 > 两位×一位，或指数底数 > 10 且指数 > 5
- 若知识点核心操作简化后仍不直观（如椭圆曲线点加），该步是否标注了**黑箱**（"这步信任计算器，关注数据流向"）？
- 示例结尾是否有一句话**缩放声明**（"真实世界把 N 换成 M 位大数，流程不变"）？
- 违规示例：RSA 示例用 `p=61, q=53`，`61×53=3233` 需要纸笔 → 违规

### M11. 重建路线图格式
第四幕空白重建是否为五列表格（# / 缺口问句 / 方案 / 解决了什么 / 露了什么）？
- 缺口列是否全部为问句？第 0 行为纯起点问题；第 1+ 行是否体现因果承接（读者能看出"为什么上一行没完"）？
- 方案列用名词短语，解决了什么列用陈述短句，露了什么列回答"为什么还不够"？
- 每格参考线 ≤ 15 字（缺口列 ≤ 30 字）？
- **因果链检验**：「露了什么」改写成问句后，能否作为下一行「缺口」的合理改写？遮住「露了什么」列时行间有跳跃感，露出来跳跃消失？
- 末行是否用 `→` 前缀标记开放缺口指向下一主题？
- 有并行分支时是否用字母后缀（5a, 5b）？

### M12. 第二幕末尾压力测试
第二幕最后一个概念消化后，是否有过渡语 + 2-3 个反事实/边界压力测试停顿点？
- 停顿点是否为真提问（`> 想一想：___？` 格式），而非 AI 自答段落？
- 违规示例：第二幕直接结束，无压力测试 → 违规
- 违规示例：写了"如果去掉 X 会怎样？"然后自己答了一段 → 违规

## 第二轮：结构判断（软建议）

以下判断需要主观评估，结果仅标记为 warning，不算 critical：

### S1. 动机缺失
某个新概念引入时，是否缺少"为什么现在需要它"的说明？
- 特别关注：引入新符号/新写法时，是否回答了"为什么不能继续用之前的写法"？

### S2. 预教嫌疑
某段引入了读者此刻不需要的概念？
- 判据：删掉这段，下一段仍然可读 → 可能出场太早

### S3. 因果链断裂
第二幕叙事是否有"跳跃"——前一个概念解决了什么、暴露了什么新问题，没有说清楚就引入了下一个概念？

### S4. 节首缺口缺失
读者不知道"为什么要读这一节"——没有制造缺口就直接陈述？

### S5. 节尾钩子缺失
一节写完就完了，没有问题或悬念拉向下一节？

### S6. 公式需求驱动缺失（仅含数学公式时检查）
公式前没有说"要解决什么问题"就直接亮公式？

### S7. 回忆锚点缺失（仅含多步推导时检查）
公式或段落引用了距离上次定义 ≥3 段的变量，却没有括号回忆提示？

## 输出格式

```json
{
  "overall_verdict": "pass | needs_revision",
  "critical_issues": [
    {
      "location": "第X段（引用段落首句）",
      "rule": "M1-M12",
      "detail": "具体描述",
      "suggestion": "一句话修改方向"
    }
  ],
  "warnings": [
    {
      "location": "第X段",
      "rule": "S1-S7",
      "detail": "描述"
    }
  ]
}
```

**只输出有问题的段落。没有问题的段落不需要记录。**

---

## 执行流程

1. 生成 Agent 完成四幕笔记
2. 启动独立审查 Agent，传入完整笔记文本
3. 审查 Agent 先跑第一轮（M1-M12），再跑第二轮（S1-S7），输出 JSON 报告
4. 生成 Agent 修复所有 `critical_issues`，酌情处理 `warnings`
5. 若有修改，再次审查（最多循环 2 次）
6. 通过后输出最终版本
