# Kingbase SSL 连接功能测试报告

**类型：** 测试报告
**测试日期：** 2026-03-24
**测试人：** yangzhao

**结论：** Kingbase（人大金仓）数据库的三种 SSL 模式（Require、Verify CA、Full Verification）均测试通过，可正常建立连接并执行查询。SSL 功能可用，可集成到生产环境。

---

## 一、测试范围与环境

> 本节回答：测什么、在哪测？

**测试目标：** 验证 Kingbase 数据库 SSL 连接的三种安全模式是否均可正常工作。

| 项目 | 值 |
|------|-----|
| 数据库类型 | Kingbase（人大金仓） |
| 服务地址 | 192.168.1.49:24321 |
| 用户名 | kingbase2 |
| 数据库名 | test |

**测试的三种 SSL 模式：**

| 模式 | 说明 | 使用的证书 |
|------|------|-----------|
| Require | 要求 SSL 加密，不验证服务端证书 | 无 |
| Verify CA | 验证服务端证书由受信 CA 签发 | root(2).crt |
| Full Verification | 验证 CA + 验证主机名/IP（SAN） | root(3).crt（SAN: DNS:example.com, IP:192.168.1.49） |

---

## 二、测试过程

> 本节回答：每种模式如何测试？

每种模式均执行以下验证步骤：
1. 使用对应 SSL 参数建立数据库连接
2. 连接成功后执行 `SELECT 1`，验证查询可正常返回

**Full Verification 模式补充说明：** root(3).crt 的 SAN 字段包含 `DNS:example.com` 和 `IP:192.168.1.49`，测试通过 IP 地址（192.168.1.49）连接时，IP SAN 匹配生效，主机名验证通过。

> 本节回答：三种模式的测试结论分别是什么？

来源：实测（环境见第一节）

---

## 四、结论

> 本节回答：测试结果对后续工作意味着什么？

三种 SSL 模式全部通过，Kingbase SSL 功能可用：

- **Require 模式**适用于仅需要加密传输、不关心服务端身份的场景。
- **Verify CA 模式**适用于需要验证服务端证书合法性的场景，需提前分发 CA 证书。
- **Full Verification 模式**是安全性最高的选项，适用于生产环境；部署时需确保证书 SAN 包含实际连接使用的 IP 或域名。
