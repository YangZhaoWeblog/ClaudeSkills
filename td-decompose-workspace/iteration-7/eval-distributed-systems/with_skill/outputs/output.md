### 🔴 学透

- **CAP定理** — "三选二"人人能背，但为什么只能二选一？不走一遍证明，后面所有设计权衡都是在背结论
  → `学透 CAP定理`
  依赖链：网络模型 → 故障模型 → **CAP定理**
  必须先学：故障模型（不理解故障类型就无法理解"分区"的精确含义）
  可跳过：网络模型（同步/异步的直觉够用就行）

- **一致性模型** — 线性一致性、顺序一致性、最终一致性不是名词列表，是一组精确定义的"系统承诺"。速读只会得到模糊印象，后面学共识、复制时全部会卡
  → `学透 一致性模型`
  依赖链：网络模型 → 时钟与因果序 → CAP定理 → **一致性模型**
  必须先学：CAP定理（它划定了一致性的可能空间）
  可跳过：时钟与因果序（"逻辑时钟给事件排序"这个直觉够了）

- **共识问题** — 为什么让 N 台机器同意一个值这么难？FLP不可能性在说什么？不理解这个，Paxos/Raft 的每一步都会觉得"为什么要这么复杂"
  → `学透 共识问题`
  依赖链：故障模型 → CAP定理 → 一致性模型 → **共识问题**
  必须先学：一致性模型（共识是实现一致性的手段，目标不清就无从理解手段）
  可跳过：故障模型和 CAP 如果前面已经学透，这里自然衔接

### 🟡 建议速读，想深学也行

- **网络模型** — 速读：同步/异步/半同步三种假设各意味着什么，*DDIA* 第8章前半。想深学 → `学透 网络模型`
- **进程与消息传递** — 速读：任意分布式系统教材第1章，有编程经验的话直觉够用
- **时钟与因果序** — 速读：Lamport 时钟论文摘要 + 向量时钟示意图。想深学 → `学透 时钟与因果序`（依赖链：网络模型 → **时钟与因果序**，先学网络模型）
- **故障模型** — 速读：崩溃故障 vs 拜占庭故障的区别，*DDIA* 第8章
- **Paxos** — 速读：Lamport 的 "Paxos Made Simple"。**强烈推荐深学** → `学透 Paxos`（依赖链：一致性模型 → 共识问题 → **Paxos**，先学共识问题）
- **复制策略** — 速读：主从 / 多主 / 无主复制的权衡，*DDIA* 第5章
- **分布式事务** — 速读：2PC 的工作流程和它为什么慢，*DDIA* 第9章

### 🟢 动手

- **Raft** — 跑 Raft 可视化（raft.github.io），然后用 Go/Rust 实现简化版 leader election + log replication
- **一致性哈希** — 手写一致性哈希 + 虚拟节点，用真实数据测分布均匀度

### 💡 适合对比着学的组合

- **CAP定理** vs **一致性模型**：CAP 是"不可能"的边界，一致性模型是"在边界内你能选什么"——先学约束再学选项，比单独学任何一个都快
- **Paxos** vs **Raft**：同一个问题的两代解法，Raft 是 Paxos 的"可理解性重构"——对比着看能同时理解"为什么难"和"怎么简化"
