## 密码学 · 知识分解 DAG

> 你提到学过对称加密和哈希，按"有一定基础"处理，这两个节点标为已掌握。

**1) 一句话学习目标**

理解密码学核心原语（对称、非对称、签名、证书）的依赖结构与设计动机，能看懂 TLS 握手中每一步在做什么、为什么。

**2) 入口与枢纽**

- 入口（无前置依赖）：对称加密 ✅、哈希函数 ✅、模运算与数论基础、随机数生成
- 核心枢纽：★ [[非对称加密]]（被 4 个节点依赖）

**3) 知识节点列表**

- [x] **[[对称加密]]** ✅ 已掌握
      用同一把密钥加密和解密的算法族（AES、ChaCha20 等）
      依赖：无
      评估：负荷 轻 · 杠杆 ★★ · 衰减 低 · 可建构 高 · 距离 近（已掌握）
      ⬜ 已掌握，作为后续节点的基础

- [x] **[[哈希函数]]** ✅ 已掌握
      将任意长输入映射为固定长摘要，不可逆且抗碰撞
      依赖：无
      评估：负荷 轻 · 杠杆 ★★ · 衰减 低 · 可建构 高 · 距离 近（已掌握）
      ⬜ 已掌握，作为后续节点的基础

- [ ] **[[随机数生成]]**
      密码学安全伪随机数生成器（CSPRNG）与熵源的区分
      依赖：无
      评估：负荷 轻 · 杠杆 ★★ · 衰减 中 · 可建构 低 · 距离 中
      🟡 建议快速阅读 支撑性知识，理解"为什么 Math.random() 不能用于密钥生成"即可

- [ ] **[[模运算与数论基础]]**
      模运算、欧拉定理、离散对数问题——非对称密码的数学地基
      依赖：无
      评估：负荷 中 · 杠杆 ★★ · 衰减 低 · 可建构 中 · 距离 中
      🟡 建议快速阅读 工具性数学，掌握性质和直觉（为什么大数分解难），不需要证明定理

- [ ] **[[分组密码工作模式]]**
      ECB/CBC/CTR/GCM——同一个分组密码如何安全地加密任意长消息
      依赖：[[对称加密]]
      评估：负荷 中 · 杠杆 ★ · 衰减 中 · 可建构 高 · 距离 近
      🔴 建议 deep-learn 从 ECB 的"企鹅图"缺陷出发，重走 CBC→CTR→AEAD 的演化路径，建构感极强，且是你已有知识的直接延伸

- [ ] **[[MAC（消息认证码）]]**
      用共享密钥验证消息完整性与来源（HMAC、CMAC 等）
      依赖：[[哈希函数]]、[[对称加密]]
      评估：负荷 轻 · 杠杆 ★ · 衰减 中 · 可建构 中 · 距离 近
      🟡 建议快速阅读 核心区分只有一个："加密 ≠ 认证"，理解这个边界即可

- [ ] **[[非对称加密]]** ★
      用公钥加密、私钥解密（或反过来），解决密钥分发问题（RSA、ElGamal）
      依赖：[[模运算与数论基础]]、[[随机数生成]]
      评估：负荷 重 · 杠杆 ★★★ · 衰减 低 · 可建构 高 · 距离 中
      🔴 建议 deep-learn 全图最高杠杆节点。从"Alice 和 Bob 如何在窃听者面前交换秘密"的困境出发，重走公钥密码学的发明之路，理解 RSA 为什么能工作

- [ ] **[[密钥交换协议]]**
      两方在不安全信道上协商出共享密钥（Diffie-Hellman）
      依赖：[[模运算与数论基础]]、[[随机数生成]]
      评估：负荷 中 · 杠杆 ★ · 衰减 低 · 可建构 高 · 距离 中
      🔴 建议 deep-learn DH 协议是密码学最优美的构造之一——"两人在广场上大声喊数字，旁听者却算不出他们的秘密"，可建构性极高

- [ ] **[[数字签名]]**
      用私钥签名、公钥验签，实现不可否认的身份证明
      依赖：[[非对称加密]]、[[哈希函数]]
      评估：负荷 中 · 杠杆 ★★ · 衰减 低 · 可建构 高 · 距离 中
      🔴 建议 deep-learn 非对称加密的镜像应用——"加密是用公钥锁、签名是用私钥盖章"，理解这个对偶关系是关键转折点

- [ ] **[[椭圆曲线密码学]]**
      用椭圆曲线上的离散对数问题替代大数分解，更短密钥达到同等安全
      依赖：[[非对称加密]]
      评估：负荷 重 · 杠杆 ★ · 衰减 低 · 可建构 低 · 距离 远
      🟡 建议快速阅读 数学门槛高且可建构性低，理解"为什么 256 位 ECC ≈ 3072 位 RSA"的直觉即可

- [ ] **[[数字证书与PKI]]**
      用可信第三方（CA）将公钥绑定到身份，构建信任链
      依赖：[[数字签名]]、[[非对称加密]]
      评估：负荷 中 · 杠杆 ★ · 衰减 中 · 可建构 中 · 距离 中
      🟢 建议直接动手 体系性知识，用 openssl 自己签发一次证书链比读十篇文章有效

- [ ] **[[TLS/SSL协议]]**
      将上述所有原语组装成端到端安全通信协议
      依赖：[[分组密码工作模式]]、[[MAC（消息认证码）]]、[[非对称加密]]、[[密钥交换协议]]、[[数字证书与PKI]]
      评估：负荷 中 · 杠杆 ★ · 衰减 中 · 可建构 低 · 距离 远
      🟢 建议直接动手 集成性协议，用 Wireshark 抓一次 TLS 1.3 握手，对着报文看每一步用了哪个原语

```
入口层         对称加密 ✅      哈希函数 ✅      模运算与数论基础      随机数生成
                │    │            │    │              │    │             │   │
                ▼    │            ▼    │              ▼    ▼             ▼   │
第二层     分组密码   └──→  MAC  ←┘    │        非对称加密 ★      密钥交换协议
           工作模式                    │         │    │    │             │
                │                     │         │    │    └→ 椭圆曲线   │
                │                     └──→ 数字签名   │      密码学     │
                │                           │        │                 │
                │                           ▼        │                 │
第三层          │                     数字证书与PKI ←─┘                 │

```

**5) 学习策略建议**

### 🔴 建议 deep-learn
- **[[非对称加密]]** — 全图杠杆最高（4 个下游节点），从密钥分发困境出发重走发明之路
- **[[分组密码工作模式]]** — 你已有对称加密基础，这是最自然的下一步；ECB→CBC→CTR→GCM 的演化路径建构感极强
- **[[密钥交换协议]]** — DH 是密码学最优美的 aha moment 之一，可建构性极高
- **[[数字签名]]** — 非对称加密的镜像应用，理解"加密 vs 签名"的对偶关系是质变点

### 🟡 建议快速阅读
- **[[模运算与数论基础]]** — 非对称加密的前置数学，掌握性质和直觉即可
- **[[随机数生成]]** — 理解 CSPRNG 与普通 PRNG 的区别，知道为什么这是安全基石
- **[[MAC（消息认证码）]]** — 核心就是"加密 ≠ 认证"这一个区分
- **[[椭圆曲线密码学]]** — 数学门槛高，理解效率优势的直觉即可

### 🟢 建议直接动手
- **[[数字证书与PKI]]** — 用 openssl 签发证书链，观察信任传递
- **[[TLS/SSL协议]]** — 用 Wireshark 抓 TLS 握手，对照 DAG 中每个原语的出场时机

### 💡 交错学习组合
- **非对称加密 ↔ 密钥交换协议**：都基于模运算，但解决不同问题（保密 vs 协商），对比学习效果最佳
- **MAC ↔ 数字签名**：都做"认证"，但一个用共享密钥一个用公私钥，对比能深化对认证本质的理解
- **分组密码工作模式 → TLS**：先 deep-learn 模式演化，再抓包看 TLS 实际选了哪个模式、为什么

**6) 建议的学习顺序**

基于你已掌握对称加密和哈希：
1. 先走左路：**分组密码工作模式**（离你最近，建构感强）
2. 快速补数学：**模运算与数论基础** + **随机数生成**（快速阅读）
3. 攻核心枢纽：**非对称加密** → **密钥交换协议**（交错学习）
4. 走签名-证书链：**数字签名** → **数字证书与PKI**（动手）
5. 收束：**TLS/SSL协议**（动手抓包，检验全局理解）
