### 🔴 学透

- **网络模型与故障模型** — 分布式系统的"物理定律"：异步网络、拜占庭故障、崩溃故障的区分决定了后面所有算法的假设边界
  → `学透 网络模型与故障模型`（无前置依赖，它是整张图的入口，所有下游节点的隐含假设都从这里来）

- **时钟与因果序** — 没有全局时钟时怎么定义"先后"？Lamport 时钟和向量时钟是理解一致性的必经之路
  → `学透 时钟与因果序`（依赖 [[网络模型与故障模型]]，它是从物理约束推导出逻辑工具的关键桥梁）

- **CAP定理** — 分布式系统的"不可能三角"，理解它为什么成立比记住结论重要十倍
  → `学透 CAP定理`（依赖 [[时钟与因果序]]、[[故障检测]]，它是枢纽——下游的一致性模型和共识问题都从这个约束出发）

- **一致性模型** — 线性一致性、顺序一致性、最终一致性不是"强弱排序"，是不同的语义合同
  → `学透 一致性模型`（依赖 [[CAP定理]]，它是全图最大枢纽(3↓)——复制策略、共识问题、分布式事务都由它定义目标）

- **共识问题** — 不是 Paxos 本身，而是"一组进程如何在有故障的情况下达成一致"这个问题的形式化定义
  → `学透 共识问题`（依赖 [[一致性模型]]，它是从"要什么一致性"到"怎么实现一致性"的转折点）

- **Paxos** — 理解 Paxos 的推导过程（而非背协议步骤）能一次性打通对所有共识算法的直觉
  → `学透 Paxos`（依赖 [[共识问题]]，它是共识算法的原型——Raft、ZAB 都是它的变体简化）

### 🟡 建议速读，想深学也行

- **故障检测** — 速读：了解心跳、超时、φ-accrual 检测器的基本思路即可。想深学 → `学透 故障检测`（依赖 [[网络模型与故障模型]]，它为共识问题提供"什么时候认为节点挂了"的判据）
- **复制策略** — 速读：主从复制、多主复制、无主复制的权衡，DERTA ch5 讲得清楚。想深学 → `学透 复制策略`（依赖 [[一致性模型]]，它把抽象的一致性保证落地为具体的数据搬运方式）
- **Raft** — 速读：Raft 论文本身就是为可读性设计的，直接读原文。想深学 → `学透 Raft`（依赖 [[共识问题]]，它是 Paxos 的工程友好版，对比着学更快）
- **分布式事务** — 速读：2PC 的流程和它为什么阻塞，3PC 为什么没解决根本问题。想深学 → `学透 分布式事务`（依赖 [[一致性模型]]，它是一致性保证在多操作场景下的延伸）
- **分片** — 速读：哈希分片 vs 范围分片，一致性哈希的基本思路。想深学 → `学透 分片`（依赖 [[复制策略]]，它解决单机放不下时数据怎么切）

### 🟢 动手

- **跑通 Raft 实现** — 用 Go 或 Rust 实现 MIT 6.824 Lab 2 的 Raft，或者跑通 hashicorp/raft 的 example，观察 leader election 和 log replication 的实际行为
- **读 etcd 源码片段** — 选 etcd 的 raftexample 目录，追踪一次 Put 请求从客户端到 committed 的完整路径，串联分片和 Raft 在生产系统中的协作

### 💡 适合对比着学的组合

- **Paxos** vs **Raft**：Raft 论文开篇就说自己是为了解决 Paxos 难懂的问题——先学 Paxos 的推导再看 Raft 的简化，能一次搞清"共识"到底在解决什么
- **线性一致性** vs **最终一致性**：不是"强 vs 弱"，是"付出多少延迟换多少确定性"——放一起对比才能理解 CAP 的真实含义
