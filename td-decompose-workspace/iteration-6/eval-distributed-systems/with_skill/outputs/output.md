### 🔴 学透

- **时钟与因果序** — 分布式系统没有全局时钟，因果序是唯一可靠的排序方式；不理解这个，后面所有一致性讨论都是空中楼阁
  → `学透 时钟与因果序`
  依赖链：网络模型 → **时钟与因果序**
  必须先学：无
  可跳过：网络模型——知道"消息可能延迟、乱序、丢失"这个直觉就够

- **一致性模型** — 线性一致 vs 因果一致 vs 最终一致的区别决定了所有后续设计选择，这是整张图的枢纽
  → `学透 一致性模型`
  依赖链：网络模型 → 时钟与因果序 → **一致性模型**
  必须先学：时钟与因果序（不知道"顺序"为什么重要，就无法区分各种一致性）
  可跳过：网络模型——有异步网络的直觉就行

- **CAP定理** — 分布式系统的"不可能三角"，理解它为什么成立比背结论重要得多
  → `学透 CAP定理`
  依赖链：网络模型 → 时钟与因果序 → 一致性模型 → **CAP定理** ← 故障模型
  必须先学：一致性模型（得知道 C 指什么）、故障模型（得知道 P 指什么）
  可跳过：网络模型、时钟与因果序——有直觉就够

- **共识问题** — 知道"让 N 个节点就一个值达成一致"为什么困难，才能理解 Paxos/Raft 解决的究竟是什么
  → `学透 共识问题`
  依赖链：网络模型 → 时钟与因果序 → 一致性模型 → **共识问题** ← 故障模型
  必须先学：一致性模型（必须先学）、故障模型（得知道节点会怎么挂）
  可跳过：网络模型、时钟与因果序——有直觉就够

- **Paxos** — 共识算法的原型，理解 Paxos 等于理解"为什么共识协议长这样"
  → `学透 Paxos`
  依赖链：网络模型 → 时钟与因果序 → 一致性模型 → 共识问题 → **Paxos**
  必须先学：共识问题（必须先学）
  可跳过：一致性模型以下——如果已经理解"为什么需要共识"就够了

- **分布式事务** — 跨分片/跨服务的原子操作，2PC/3PC/Saga 各有取舍，工程中绕不开
  → `学透 分布式事务`
  依赖链：一致性模型 → **分布式事务** ← 复制状态机
  必须先学：一致性模型（必须先学）、复制状态机（至少速读——得知道单分片内怎么保证一致）
  可跳过：Paxos 的具体细节——了解"共识能做到"就够

### 🟡 建议速读，想深学也行

- **网络模型** — 速读：DDIA 第8章前半部分，抓住同步/异步/半同步的区别。想深学 → `学透 网络模型`（无前置依赖，直接开始）
- **故障模型** — 速读：DDIA 第8章故障分类，抓住 crash-stop vs Byzantine 的区分。想深学 → `学透 故障模型`（无前置依赖）
- **Raft** — 速读：raft.github.io 的可视化动画足够建立直觉。想深学 → `学透 Raft`（依赖链：一致性模型 → 共识问题 → **Raft**，必须先学共识问题）
- **复制状态机** — 速读：理解"所有副本按相同顺序执行相同命令 = 状态一致"即可。想深学 → `学透 复制状态机`（依赖链：共识问题 → Paxos/Raft → **复制状态机**，必须先学至少一种共识算法）
- **数据分片与一致性哈希** — 速读：DDIA 第6章，抓住"数据怎么切、切完怎么找"。想深学 → `学透 数据分片与一致性哈希`（依赖链：网络模型 → **数据分片与一致性哈希**，网络模型有直觉就够）
- **最终一致性与CRDT** — 速读：Martin Kleppmann 的 CRDT 入门博客。想深学 → `学透 最终一致性与CRDT`（依赖链：一致性模型 → CAP定理 → **最终一致性与CRDT**，必须先学一致性模型和 CAP定理）

### 🟢 动手

- **搭一个分布式KV** — 用 Go/Rust 实现基于 Raft 的分布式 KV store（推荐 MIT 6.824 lab 系列），把复制、分片、容错串起来

### 💡 适合对比着学的组合

- **Paxos** vs **Raft**：同一个问题的两种解法，对比能看清 Raft "更容易理解"背后牺牲了什么、换来了什么
- **分布式事务** vs **最终一致性与CRDT**：强一致和弱一致的两个极端策略，放一起学能看清 CAP 取舍在工程中怎么落地
