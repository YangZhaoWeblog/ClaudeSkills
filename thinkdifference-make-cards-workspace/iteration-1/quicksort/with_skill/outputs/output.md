[卡片1] how
Q: 快速排序每一层递归的执行步骤是什么？（挖空）
   选定 pivot → ___ → 对左右子数组 ___
A: 分区（Partition）：把比 pivot 小的元素移到左边，比 pivot 大的移到右边；
   然后对左右子数组递归执行同样的过程。

[卡片2] why
Q: 快速排序的平均时间复杂度为什么是 O(n log n)，而不是 O(n^2)？
A: 理想情况下每次选的 pivot 将数组均分为两半，递归深度约为 log n，
   每层都需要 O(n) 的分区操作，总计 O(n log n)。
   O(n^2) 是极端情况，平均随机分布下不会出现。

[卡片3] 果
Q: 快速排序在什么具体情况下会退化到 O(n^2)？
A: 当数组已经有序（升序或降序），且每次固定选第一个（或最后一个）元素作为 pivot 时。
   每次分区只能分出 0 个和 n-1 个元素，递归深度变为 n 层，总操作数为 O(n^2)。

[卡片4] reverse
Q: 快速排序的空间复杂度是 O(log n)，但它被称为"原地排序（in-place）"——这不矛盾吗？
A: 不矛盾。"原地"指不使用额外的数组来存储元素，分区操作是通过原数组内的元素交换完成的。
   O(log n) 的空间消耗来自递归调用栈，不是额外数组。
   所以快排是原地的（不需要额外数组），但并非零空间开销。

[卡片5] 场景
Q: 面试官问："快排和归并排序，你会在什么情况下分别选择哪个？"
   核心判断依据是什么？
A: 核心判断依据是：是否需要稳定排序，以及内存是否敏感。
   - 需要稳定性（如排序含多键值的对象）-> 选归并排序（Merge Sort）
   - 内存敏感、允许不稳定 -> 选快速排序（in-place，额外空间仅 O(log n)）
   注：归并排序需要 O(n) 的额外空间。

[卡片6] why
Q: 快速排序为什么是不稳定（unstable）的排序算法？
A: 分区过程中，相等元素的相对顺序可能因交换操作而改变。
   例如，若两个相等的元素 A 和 B（A 在 B 前）恰好在 pivot 两侧，
   交换后 B 可能排在 A 前面。稳定性需要额外记录位置信息才能保证。

[卡片7] how（三数取中法）
Q: 三数取中法（Median-of-Three）是如何避免快速排序最坏情况的？
A: 取数组的首、中、尾三个元素，选其中值（median）作为 pivot。
   这样即使数组有序，也不会每次都选到最小或最大值，
   从而避免分区极度不均匀，降低退化到 O(n^2) 的概率。

[卡片8] 果（优化策略）
Q: 快速排序中，对小数组切换到插入排序（Insertion Sort）的理由是什么？
A: 当子数组规模很小（如 n < 10~20）时，递归调用的函数开销相对于排序本身比例很大。
   插入排序在小数组上常数因子更小、无递归开销，实际运行更快。
   这是工程实现中的常见优化，标准库（如 Java 的 Arrays.sort）通常内置此策略。

[卡片9] 连接
Q: 快速排序（Quick Sort）和堆排序（Heap Sort）都能达到 O(n log n)，
   实际工程中快排更常用，原因是什么？
A: 快排的常数因子更小，缓存局部性（cache locality）更好：
   分区操作是在连续内存上线性扫描，CPU 缓存命中率高。
   堆排序虽然最坏也是 O(n log n) 且原地，但堆的随机访问模式对缓存不友好，
   实际运行往往比快排慢 2~3 倍。
