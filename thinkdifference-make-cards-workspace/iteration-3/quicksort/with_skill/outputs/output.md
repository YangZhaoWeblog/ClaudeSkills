### 核心机制

**Q:** 在快速排序的核心思想中，"分治"具体分为哪三步？

**A:** (1) **分**：选一个基准元素（pivot）；(2) **治**：将数组分区——比 pivot 小的放左边，比 pivot 大的放右边；(3) **递归**：对左右两部分各自重复上述过程，直到子数组长度为 0 或 1。

---

**Q:** 在快速排序的分区操作（partition）中，一次分区完成后，pivot 处于什么位置？

**A:** pivot 处于它在最终排序结果中的正确位置——左边所有元素都不大于它，右边所有元素都不小于它。这意味着每次分区至少确定一个元素的最终位置。

---

**Q:** 在快速排序的执行流程中：选 pivot → ___ → 递归左半部分 → 递归右半部分。空缺处是什么操作？

**A:** 分区（partition）——遍历数组，将比 pivot 小的元素移到左边，比 pivot 大的元素移到右边，pivot 放到最终位置。

---

### 复杂度分析

**Q:** 在快速排序的平均情况下，为什么时间复杂度是 O(n log n)？

**A:** 平均情况下，每次 pivot 将数组大致对半分，递归深度为 log n 层；每层需要遍历所有 n 个元素做分区操作。总工作量 = 每层 O(n) x log n 层 = O(n log n)。

---

**Q:** 在快速排序的最坏情况下，时间复杂度退化为 O(n²)，这是什么条件触发的？

**A:** 每次分区都极度不均匀——一边 n-1 个元素，另一边 0 个元素。典型触发场景：数组已排序（或逆序），且每次都选第一个（或最后一个）元素作为 pivot。此时递归深度退化为 n 层，每层仍需 O(n) 工作。

---

**Q:** 在快速排序中，空间复杂度 O(log n) 消耗在哪里？

**A:** 消耗在递归调用栈上。平均情况下递归深度为 log n，每层栈帧占常数空间，因此总空间 O(log n)。注意：最坏情况下递归深度退化为 n，空间复杂度也退化为 O(n)。

---

### Reverse 角度

**Q:** 在排序算法选择中，如果观察到一个排序算法在已排序数组上性能急剧下降到 O(n²)，最可能是哪种算法？为什么？

**A:** 最可能是快速排序（使用首元素或末元素作为 pivot）。因为已排序数组导致每次分区极度不均匀（一边 n-1，一边 0），递归深度退化为 n，总时间 n + (n-1) + ... + 1 = O(n²)。

---

**Q:** 在快速排序的分析中，如果希望最坏情况不出现 O(n²)，需要从根本上解决什么问题？

**A:** 需要避免 pivot 反复选到极端值（最大或最小元素）。根本问题是 pivot 选择策略——只要能保证 pivot 大致位于中间位置（如三数取中法、随机选取），就能避免最坏情况。

---

**Q:** 在排序算法的对比中，如果一个场景要求排序必须是稳定的，快速排序为什么不能直接使用？应该选什么替代？

**A:** 快速排序在分区过程中会交换不相邻的元素，导致相同值的元素可能改变相对顺序，因此是不稳定排序。应选归并排序（Merge Sort）——它在合并时可以保证相同值元素的相对顺序不变，是稳定的。

---

### 与归并排序的对比

**Q:** 在排序算法的对比中，快速排序和归并排序都是 O(n log n) 的分治算法，它们在空间使用上的核心区别是什么？

**A:**

```
             快速排序          归并排序
空间需求    O(log n)           O(n)
            (仅递归栈)         (需要额外数组做合并)
原地排序?    是                 否
```

快排在分区时通过元素交换实现原地操作；归并在合并时必须借助临时数组拷贝元素。

---

**Q:** 在排序算法的对比中，快速排序是"不稳定排序"，"不稳定"具体指什么？

**A:** 指相等的元素在排序后可能改变它们在原数组中的相对顺序。例如两个值相同的记录 A 和 B，排序前 A 在 B 前面，排序后 B 可能跑到 A 前面。

---

### 优化策略

**Q:** 在快速排序的 pivot 选择优化中，"三数取中法"（median-of-three）是怎么做的？它解决什么问题？

**A:** 取数组首、中、末三个元素的中位数作为 pivot。解决的问题：避免在已排序或逆序数组上总是选到极端值，从而避免 O(n²) 的最坏情况。

---

**Q:** 在快速排序的优化中，为什么小数组要切换到插入排序（Insertion Sort）而不是继续递归？

**A:** 小数组（通常 n < 10~20）时，快排的递归调用开销（函数调用、栈帧分配）相对于数据量来说占比过大。插入排序虽然是 O(n²)，但在小 n 下常数因子极小、缓存友好，实际速度更快。

---

**Q:** 在快速排序的优化中，"尾递归优化"是如何减少栈深度的？

**A:** 对分区后的两个子数组，先递归较小的那个，然后用循环（而非递归）处理较大的那个。这样递归深度最多为 O(log n)，即使在最坏分区情况下也能控制栈深度，避免栈溢出。

---

## 3. 补充卡片

---

**Q:** 在快速排序的实际应用中，为什么很多标准库（如 C 的 qsort、Java 的 Arrays.sort 对基本类型）选择快速排序而非归并排序？

**A:** 快速排序是原地排序，不需要额外 O(n) 空间；且平均情况下的常数因子较小（缓存局部性好）。对于不需要稳定性的基本类型排序，快排在实际运行中通常比归并排序更快。

---

**Q:** 在快速排序的复杂度分析中，平均 O(n log n) 和最坏 O(n²) 的根本区别取决于什么单一因素？

**A:** pivot 的选择质量。pivot 越接近中位数，分区越均匀，越接近 O(n log n)；pivot 越极端（最大/最小值），分区越不均匀，越接近 O(n²)。一切优化的本质都是在改善 pivot 的选择。
